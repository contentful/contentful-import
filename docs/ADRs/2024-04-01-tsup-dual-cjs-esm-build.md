# Use tsup for Dual CJS+ESM Build Output

## Status

Accepted

## Context

`contentful-import` originally used Babel for transpilation, targeting ES5/CommonJS output. As the npm ecosystem moved toward ESM, the package needed to support both `require()` (CommonJS) and `import` (ESM) consumers. The team also migrated the source to TypeScript, requiring a build tool that could compile `.ts` files and emit type declarations.

The challenge: the package depends on `cli-table3`, which requires `esModuleInterop: true` in TypeScript config. But setting `esModuleInterop: true` can break direct `require('contentful-import')` calls (without `.default`) for CommonJS consumers. A dual-output build was needed to resolve this tension without breaking the public API.

Source: `1fc7100 fix: remove unused babel libs` (Babel removal), `cb0f054 build(deps-dev): bump tsup` (tsup adoption). See also the inline comment in `lib/index.ts` explaining the dual-export workaround.

## Decision

Use `tsup` to produce both CJS (`dist/index.js`) and ESM (`dist/index.mjs`) outputs from a single TypeScript source. The `lib/index.ts` exports via both `export default` (ESM) and `module.exports = ...` (CJS) simultaneously as an explicit workaround for the `esModuleInterop`/`cli-table3` constraint.

```json
"tsup": {
  "entry": ["lib/index.ts", "lib/usageParams.ts"],
  "format": ["cjs", "esm"],
  "clean": true,
  "dts": true
}
```

The `package.json` `exports` map directs Node.js and bundlers to the correct format.

## Consequences

- Consumers using `require('contentful-import')` get the CommonJS build directly (no `.default` needed)
- Consumers using `import contentfulImport from 'contentful-import'` get the ESM build
- The dual-export pattern in `lib/index.ts` is non-standard and must not be changed without understanding the `cli-table3` constraint
- Babel and related `@babel/*` dependencies were removed
- `ts-jest` is used in tests (not `tsup`) to run TypeScript test files via Jest
- Type declarations (`dist/index.d.ts`) are generated by tsup alongside the JS outputs
